> **Swearing Online: Exploring Dimensions of Variability in Born-Digital Data**  
> Martin Schweinberger et al.  
> *Submitted to Digital Scholarship in the Humanities* (Manuscript ID: DSH-2025-0218)

---

## Overview

This paper investigates the linguistic and sociolinguistic dimensions of variation in swearing and vulgarity in born-digital online discourse. Using large-scale corpus data and computational methods, we examine how swear words and taboo language vary systematically across registers, regions, and online contexts. The analyses draw on the computational and quantitative approaches characteristic of the Digital Humanities, bridging corpus linguistics, sociolinguistics, and data science.

The repository is structured to support full reproducibility of all analyses, figures, and results reported in the paper.

## Repository Structure

```
SwearingOnline_DSH-2025-0218/
├── scripts/               # R notebooks and analysis scripts
├── figures/               # Output figures generated by the analysis scripts
├── images/                # Supporting images used in the notebooks or paper
├── renv/                  # renv lockfile and infrastructure for reproducibility
├── .Rprofile              # R project profile (activates renv automatically)
├── SwearingOnline_DSH-2025-0218.Rproj  # RStudio project file
└── .gitignore
```

### Key Directories

- **`scripts/`** — R notebooks (`.Rmd` / `.html`) containing all data processing, analysis, and visualisation code. Each notebook corresponds to a stage of the analysis pipeline described in the paper.
- **`figures/`** — All figures produced by the analysis scripts and included in the manuscript.
- **`images/`** — Ancillary images used in the notebooks or supplementary materials.
- **`renv/`** — Package snapshot managed by [`renv`](https://rstudio.github.io/renv/) to ensure package versions are locked for reproducibility.

## Reproducibility

This project uses [`renv`](https://rstudio.github.io/renv/) to manage R package dependencies. All package versions are recorded in `renv.lock`, so you can restore the exact computational environment used in the paper.

### Requirements

- [R](https://www.r-project.org/) (≥ 4.1.0 recommended)
- [RStudio](https://posit.co/download/rstudio-desktop/) (recommended for working with `.Rproj` and R Markdown notebooks)

### Getting Started

1. **Clone this repository**

   ```bash
   git clone https://github.com/MartinSchweinberger/SwearingOnline_DSH-2025-0218.git
   cd SwearingOnline_DSH-2025-0218
   ```

2. **Open the RStudio project**

   Open `SwearingOnline_DSH-2025-0218.Rproj` in RStudio. The `.Rprofile` will automatically activate `renv`.

3. **Restore the package environment**

   In the R console, run:

   ```r
   renv::restore()
   ```

   This will install all required packages at the exact versions used in the analysis.

4. **Run the analysis notebooks**

   Open and knit the R Markdown scripts in the `scripts/` folder in order. Rendered HTML outputs and figures will be written to `figures/`.
